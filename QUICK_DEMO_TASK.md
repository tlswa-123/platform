# 快速验证 Demo —— 任务说明

## 目标

验证「AI 一句话生成可玩游戏」的端到端效果。不需要 WeCreat 的 UI 外壳，直接在 WorkBuddy 里用 Agent 能力生成一个完整可玩的 HTML5 小游戏。

## 核心思路

你（WorkBuddy Agent）本身就具备：
- 代码生成能力（写 HTML/JS/CSS）
- 图片生成能力（image_gen 工具）
- 音效生成能力（sound-effects skill）
- 文件系统操作能力（创建目录、写文件）
- 本地服务器启动能力（python http.server / npx serve）
- 浏览器预览能力（preview_url）

所以你自己就是那个「AI 游戏创作工具」的 MVP 验证。

## 具体任务

### 1. 选一个游戏做验证

从以下候选中选一个（推荐第一个，最简单也最能展示效果）：

- **贪吃蛇**：经典玩法，360度自由转向版，彩色食物，简洁UI
- **打砖块**：挡板+弹球+彩色砖块阵列，简单物理
- **太空射击**：自动射击，敌机波次，道具掉落

### 2. 生成游戏素材

用 image_gen 工具生成游戏需要的视觉素材：
- 游戏背景图（如果需要）
- 角色/元素素材（如果用精灵而非代码绘制）
- UI 元素（开始按钮、Logo 等）

用 sound-effects skill 生成音效：
- 背景音乐（可选）
- 交互音效（吃食物、碰撞、得分等）

### 3. 编写完整可玩游戏

技术栈：**单文件 index.html**（内联 CSS + JS），不要用任何框架。

要求：
- 完整游戏循环：开始界面 → 游戏中 → 结束界面 → 重新开始
- 触控+键盘双操控（移动端适配）
- 计分系统 + localStorage 最高分
- 流畅 60fps
- 游戏素材内嵌或放在同目录 assets/ 下

### 4. 预览验证

用 preview_url 展示最终效果，确保游戏可玩。

## 关键约束

1. **不要用 Phaser 等游戏框架**，用原生 Canvas 2D API
2. **单文件优先**，尽量把 JS 和 CSS 都内联在 index.html 里
3. **素材宁少勿多**，优先用代码绘制（圆、方块、渐变），AI 生成的图片素材作为锦上添花
4. **先保证可玩性**，再考虑美观度
5. 生成完成后用 `python3 -m http.server 8900` 启动本地服务，然后用 preview_url 展示

## 成功标准

- [ ] 游戏能正常加载并开始
- [ ] 核心玩法可以正常游玩
- [ ] 有计分系统
- [ ] 有开始/结束界面
- [ ] 60fps 流畅运行
- [ ] 有至少 2 个 AI 生成的音效
